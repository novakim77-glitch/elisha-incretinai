# 기술소개서 — 특허 출원 #2
## IMEM-DM (Incretin Metabolic Efficiency Model — Diabetes Management) — 2형 당뇨 관리

**발명의 명칭 (안)**: 혈당 변동성·약물 보정·스트레스 계수를 통합한 인크레틴 기반 당뇨 관리 시스템 및 방법

**출원인 (예정)**: [회사명 / 대표자명]
**제출일**: 2026년 4월 ~ 5월 (특허 #1과 1~4주 간격 권장)

> **특허 #1과의 관계**: 본 발명은 특허 #1(IMEM 비만·체중 관리)의 α·β·γ 3계수 구조를 기반으로, 당뇨 적응증에 특화된 3개의 신규 계수(δ·ε·ζ)를 추가하여 6계수 확장 시스템을 제안합니다. **국내우선권(특허법 제55조) 주장을 통해 특허 #1 출원 후 1년 이내 출원 시 특허 #1의 우선일 이익을 수혜 가능합니다.**

## 1. 기술 개요

### 1.1 발명의 배경

2형 당뇨 환자는 GLP-1 분비 능력이 손상되어 있으며, GLP-1 수용체 작용제(GLP-1 RA) 또는 DPP-4 억제제 등 약물로 이를 보완합니다. 현재 디지털 헬스 도구들은 혈당 기록(CGM 연동 포함)과 식단·운동 추적을 제공하지만, 아래의 3가지 임상적 갭을 해결하지 못합니다:

1. **혈당 변동성의 정량적 통합 부재**: TIR(목표 범위 내 시간), GVI(Glycemic Variability Index), MAGE(Mean Amplitude of Glycemic Excursion) 같은 당뇨 표준 지표가 행동 권고와 연결되지 않음
2. **약물 효과 미보정**: 메트포르민·GLP-1 RA·SGLT-2i·인슐린 등 약물이 혈당과 생활습관 지표에 미치는 효과를 제거하지 않으면 모델 점수가 약물 효과를 생활습관 개선으로 오귀인함
3. **저혈당 안전성 로직 부재**: 인슐린·SU 계열 약물 복용 중 과도한 단식이나 고강도 운동은 저혈당 위험을 초래할 수 있으나, 기존 앱은 이를 차단하는 안전 레이어를 갖추지 않음

### 1.2 핵심 공식

```
Efficiency_DM = α × β × γ × δ × ε × ζ

  α (Alpha) : 일주기 생체리듬 지수       [특허 #1에서 계승 + 혈당 리듬 보정 확장]
  β (Beta)  : 영양 시퀀스 효율 지수      [특허 #1에서 계승 + 저혈당지수 식품 가중치 추가]
  γ (Gamma) : 신체 활동 지수            [특허 #1에서 계승 + 혈당 강하 안전 창 추가]
  δ (Delta) : 혈당 변동성 역지수         [신규 — 당뇨 특화]
  ε (Epsilon): 약물·복약 보정 계수       [신규 — 당뇨 특화]
  ζ (Zeta)  : 스트레스·코르티솔 보정 계수 [신규 — 당뇨 특화]
```

## 2. 신규 계수 상세 — 발명의 핵심

### 2.1 δ (Delta) — 혈당 변동성 역지수 [핵심 신규 발명]

**목적**: 혈당 안정성을 정량화하여 생활습관 점수에 통합

**입력 데이터**:
- CGM(연속혈당측정기) 데이터 스트림 (API: Dexcom/Libre 연동)
- 자가혈당(SMBG) 수동 입력 (4~7회/일)

**산출 지표 및 가중치**:

| 지표 | 설명 | 가중치 |
|------|------|--------|
| **TIR** (Time in Range, 70~180 mg/dL) | 목표 범위 내 체류 시간 비율 | 40% |
| **GVI** (Glycemic Variability Index) | 혈당 곡선의 변동 복잡도 | 25% |
| **MAGE** (Mean Amplitude) | 평균 혈당 진폭 (mg/dL) | 20% |
| **CV** (Coefficient of Variation) | 혈당 표준편차/평균 비율 | 10% |
| **Low BG AUC** | 저혈당 구간(< 70 mg/dL) 누적 면적 | 5% (위험 가중) |

**δ 공식**:
```
δ = TIR_norm × 0.4 + (1 - GVI_norm) × 0.25 + (1 - MAGE_norm) × 0.20
    + (1 - CV_norm) × 0.10 − LowAUC_penalty × 0.05

범위: 0.60 ~ 1.00  (최저는 안전 하한 설정으로 0 도달 방지)
```

**신규성**: 기존 CGM 앱(Dexcom Clarity, LibreView)은 혈당 변동 지표를 **시각화만** 하며, 생활습관 행동 점수에 **수학적으로 통합**하지 않습니다.

### 2.2 ε (Epsilon) — 약물·복약 보정 계수 [핵심 신규 발명]

**목적**: 약물 효과를 Efficiency_DM에서 분리하여 순수 생활습관 기여도 추정

**약물 분류 및 보정 값**:

| 약물 군 | 분류 기호 | ε 범위 | 보정 방향 |
|---------|----------|--------|----------|
| **메트포르민** | Met+ | 0.95 ~ 0.98 | β 과대평가 소폭 차감 |
| **GLP-1 RA** (세마글루타이드 등) | GLP+ | 0.85 ~ 0.92 | α·β 효과 과대평가 차감 |
| **SGLT-2i** (다파글리플로진 등) | SGLT+ | 0.88 ~ 0.94 | δ 상향 과보정 차감 |
| **인슐린 (기저)** | Ins-B | 0.80 ~ 0.88 | α·δ 복합 차감 |
| **인슐린 (식후)** | Ins-R | 0.75 ~ 0.85 | β·δ 복합 차감 + 저혈당 위험 가중 |
| **DPP-4 억제제** | DPP+ | 0.93 ~ 0.97 | 소폭 차감 |
| **약물 미복용** | None | 1.00 | 보정 없음 |

**복약 순응도 통합**:
```
ε_adjusted = ε_base × 복약_순응도_비율 (0.0~1.0)
복약 순응도: 텔레그램 봇 "약 복용 완료" 체크 기반 7일 이동 평균
```

**신규성**: 약물 효과를 계수로 수학적으로 분리하여 생활습관 기여도만 정량화하는 알고리즘은 기존 당뇨 디지털 헬스 도구에 존재하지 않습니다.

### 2.3 ζ (Zeta) — 스트레스·코르티솔 보정 계수 [신규 발명]

**목적**: 스트레스 호르몬(코르티솔)이 혈당 및 인슐린 저항성에 미치는 영향 보정

**입력 데이터**:
- 웨어러블 HRV(심박변이도) — 교감/부교감 균형 대리 지표
- 수면 질 점수 (수면 분절, 렘수면 비율)
- 사용자 자가보고 스트레스 레이팅 (1~5점)

**ζ 산출**:
```
ζ = 0.60 + HRV_norm × 0.20 + SleepQuality_norm × 0.15 + SelfReport_norm × 0.05

범위: 0.60 ~ 1.00  (고스트레스 → ζ 낮음 → Efficiency_DM 하향)
```

**특수 이벤트 감지**:
- **새벽 현상(Dawn Phenomenon)**: 새벽 3~5시 혈당 상승 감지 → α 계수 자동 조정
- **소모기 효과(Somogyi Phenomenon)**: 야간 저혈당 후 반동 혈당 상승 패턴 인식 → 경고 및 인슐린 용량 재검토 권고

## 3. 시스템 구성 — 추가 신규 요소

### 3.1 3단계 모드 활성화 (Mode Activation System) [신규 발명]

당뇨 환자의 디지털 헬스 리터러시와 의료 접근성이 다양하므로, 계수 활성화를 단계적으로 구현:

| 모드 | 활성 계수 | 입력 데이터 요건 | 대상 사용자 |
|------|----------|----------------|-----------|
| **Basic** | α × β × γ (3계수) | 자가 입력만 | 디지털 초보·기기 없음 |
| **Standard** | α × β × γ × δ_SMBG × ε × ζ | 자가혈당 4~7회/일 | 일반 2형 당뇨 환자 |
| **Full** | α × β × γ × δ_CGM × ε × ζ | CGM + 웨어러블 | 적극적 자기관리자 |

사용자 모드는 초기 온보딩 시 선택, 이후 기기 연동 추가 시 자동 업그레이드.

### 3.2 혈당 시계 (Glycemic Clock) — Bio-Sync DM [신규 발명]

특허 #1의 Bio-Sync 타이머를 당뇨 임상 기준으로 확장:

| 시간대 | IMEM-DM 이벤트 | 의학적 근거 |
|--------|--------------|-----------|
| 06:00 | 공복혈당 측정 창 | 새벽 현상 모니터링 |
| 07:00 | 아침 식전 약 복용 확인 | 복약 순응도 |
| 07:30~08:30 | 식후 혈당 상승 예측 알림 | 아침 식사 GI 관리 |
| 10:00 | 혈당 안정 확인 + 가벼운 활동 권고 | 식후 운동 효과 |
| 11:30 | 점심 준비 혈당 코칭 | 저혈당 예방 점심 설계 |
| 13:00~14:00 | 식후 혈당 피크 모니터링 | MAGE 측정 창 |
| 16:30 | 저혈당 위험 창 경고 | 인슐린 사용자 |
| 19:00 | 저녁 식사 혈당 관리 | 야식 R-06 + 새벽 현상 연계 |
| 22:00 | 취침 전 혈당 + 리캡 | 야간 저혈당 예방 기준 |

**신규성**: 13개 이벤트(vs. IMEM 비만 3~4개)를 24시간에 걸쳐 혈당 관리 목적으로 배치하는 알고리즘.

### 3.3 안전성 레이어 (Safety Layer) [신규 발명]

모든 생활습관 권고가 실제 사용자에게 표시되기 전 통과해야 하는 안전 필터:

```
Safety Check Pipeline:
사용자 프로필 (약물 종류, 기저질환, 최근 혈당 이력)
          ↓
[규칙 기반 필터]
  - 저혈당 위험: 혈당 < 90 mg/dL 이면 단식·운동 중강도 이상 차단
  - 약물 상호작용: 인슐린+고강도 운동 조합 경고
  - 새벽 현상: 공복혈당 > 140이면 단식 시간 단축 자동 조정
          ↓
[AI 코치 전달 → 표시]
```

**신규성**: 생활습관 코칭 AI가 권고를 생성하기 전 안전 필터를 거치는 파이프라인은 당뇨 디지털 헬스 분야에서 표준화된 구현이 없습니다.

### 3.4 Calibration Profile 시스템 [신규 발명]

당뇨 표준 임상 지표로 α·β·γ 계수를 재보정:

| 보정 기준 | 방법 |
|----------|------|
| HbA1c 3개월치 | α 계수 기준선 재설정 |
| 약물 변경 이력 | ε 계수 재매핑 |
| 체중·신체 활동 수준 | γ 계수 민감도 재조정 |

임상 검사 결과를 앱에 입력 또는 EMR 연동(향후) 시 자동 재보정.

## 4. 종래 기술과의 차별점

| 구분 | 종래 기술 | 본 발명 (IMEM-DM) |
|------|----------|-----------------|
| **CGM 활용** | 시각화 + 경고 | δ 계수로 변환 → 생활습관 점수 통합 |
| **약물 영향** | 미반영 (혼재) | ε 계수로 수학적 분리 |
| **스트레스** | 미반영 | ζ 계수 (HRV + 수면 + 자가보고) |
| **안전성** | 없음 | 3단 Safety Layer (권고 전 필터) |
| **모드** | 단일 구성 | Basic/Standard/Full 3단 모드 |
| **시간 관리** | 2~4개 알림 | 24h 9개 혈당 시계 이벤트 |
| **특허 #1 연계** | 독립 시스템 | 계수 구조 계승 + 3계수 확장 (Parent-Child) |

**핵심 신규성 주장**:
1. 혈당 변동성(TIR·GVI·MAGE·CV)을 단일 δ 계수로 통합하는 수학적 모델
2. 약물 종류·복약 순응도를 ε 계수로 정량화하여 생활습관 기여도만 분리하는 알고리즘
3. 스트레스/코르티솔을 HRV·수면·자가보고 합성으로 ζ 계수화하는 방법
4. 6계수 곱산 모델에서 약물·기기 데이터 가용성에 따라 3단계로 활성화되는 모드 전환 시스템
5. 생활습관 AI 코칭 권고가 표시되기 전 저혈당·약물 상호작용 안전 필터를 거치는 파이프라인

## 5. 청구 범위 후보 (초안)

**독립항 후보**
1. α·β·γ·δ·ε·ζ 6계수의 곱산으로 당뇨 환자의 인크레틴 대사 효율을 산출하는 방법
2. CGM/SMBG 데이터에서 TIR·GVI·MAGE·CV를 추출하여 단일 δ 계수로 변환하는 방법
3. 약물 종류 및 복약 순응도를 ε 계수로 수학적으로 정량화하여 생활습관 점수를 보정하는 방법
4. 생활습관 권고 전 저혈당 위험·약물 상호작용을 검사하는 안전 레이어 파이프라인

**종속항 후보**
5. HRV·수면 질·자가보고를 합성한 ζ 스트레스 보정 계수 산출 방법
6. 기기 가용성에 따라 Basic/Standard/Full로 계수 활성화를 전환하는 모드 시스템
7. 24시간 혈당 이벤트(Glycemic Clock)를 기반으로 한 프로액티브 알림 시스템
8. 새벽 현상 및 소모기 효과를 자동 감지하여 α 계수를 동적 조정하는 방법
9. Calibration Profile을 통해 HbA1c·약물 변경 이력으로 계수를 재보정하는 방법


| **Safety Layer** | 설계 완료, 미구현 |

> **설계 완료·미구현 상태**: 공개되지 않은 내부 설계이므로, 신규성 요건 충족. 특허 출원 후 개발 착수 계획.

## 8. 변리사 검토 요청 사항

1. **선행 기술 서치**: δ·ε·ζ 각 계수의 당뇨 디지털 헬스 선행 기술 조사 (CGM 기반 점수화, 약물 보정 알고리즘)
2. **특허 #1 우선권 활용**: 국내우선권(제55조) 주장 조건 및 이중 출원 금지 조항 검토
3. **청구항 분할 전략**: α·β·γ 공유 부분과 δ·ε·ζ 신규 부분의 청구항 구분 방법
4. **Safety Layer 보호 방법**: SW 안전 파이프라인의 특허 보호 가능성 검토 (의료기기 SW 기준 고려)
5. **의료기기 해당 여부**: δ·ε 계수 기반 권고가 의료기기법(SaMD) 적용 범위에 해당하는지 사전 검토 필요
