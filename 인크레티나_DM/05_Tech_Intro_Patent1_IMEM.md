## 1. 기술 개요

### 1.1 발명의 배경

GLP-1(Glucagon-Like Peptide-1) 기반 치료 원리는 인슐린 분비 자극, 식욕 억제, 위 배출 지연, 보상 회로 둔화의 복합 경로를 통해 작동합니다. 기존 디지털 헬스 앱은 칼로리 계산 또는 단순 루틴 체크에 그치며, **이러한 인크레틴 호르몬의 다축 생리 기전을 실시간으로 모델링하는 알고리즘을 갖추지 못했습니다**.

본 발명은 GLP-1 자가분비를 생활습관으로 최적화하는 원리를 3개의 계량 계수(α, β, γ)로 수학적으로 표현하고, 사용자 개인 데이터를 실시간으로 반영하는 **IMEM(Incretin Metabolic Efficiency Model)**을 제안합니다.

### 1.2 핵심 공식

```
Efficiency = α × β × γ

  α (Alpha) : 일주기 생체리듬 지수  — 수면·공복·시간대 정렬
  β (Beta)  : 영양 시퀀스 효율 지수 — 식사 순서·칼로리 밀도·β_meal 보정
  γ (Gamma) : 신체 활동 지수       — 운동 타이밍·강도·걷기 효과

β_net = β_base × β_meal − Σpenalty + Σrecovery
β_meal = 0.95 + avgBetaScore × 0.075   (범위: 0.95 ~ 1.025)
```

각 계수는 0.0~1.0 범위로 정규화되며, 최종 Efficiency는 실시간 체중 변화 예측 및 행동 코칭의 근거로 사용됩니다.

## 2. 발명의 주요 구성 요소

### 2.1 IMEM 점수 산출 엔진

| 구성 | 설명 |
|------|------|
| **α 계수 모듈** | 수면 시작·종료 시간, 14시간 단식 구간, 식후 산책 타이밍을 입력받아 일주기 정렬도를 0.0~1.0으로 정량화 |
| **β 계수 모듈** | 채소→단백질→탄수화물 섭취 순서, 끼니별 칼로리 비중(아침 25%·점심 40%·저녁 30%), 야식 페널티를 결합한 영양 효율 계산 |
| **β_meal 서브모듈** | 식사 기록(텍스트/사진)으로부터 매크로 영양소를 추출, 식사 품질 점수를 0.95~1.025 범위의 실수로 매핑하여 β에 곱산 보정 |
| **γ 계수 모듈** | 식후 40분 내 걷기, 운동 강도·지속 시간을 가중 합산하여 신체 활동의 인크레틴 분비 증폭 효과를 수치화 |
| **위험 페널티 시스템** | 8종 위험 행동(R-01~R-08: 야식·수면 부족·단식 중단·고당지수 식품 등)에 계수별 차감값 적용, 회복 미션 이행 시 부분 복원 |
| **IMEM 점수 통합** | α×β×γ 곱을 0~100점 선형 변환, 4주 체중 변화 예측 회귀 모델과 연계 |

### 2.2 Bio-Sync 타이머 시스템

- **골든타임 알림**: α 계수 최적화를 위한 시간대별 행동 창(창 = Window) 계산 및 알림
  - 14h 공복 구간 (저녁 종료~익일 아침)
  - 식후 산책 창 (식사 후 20~60분)
  - 수면 준비 창 (목표 취침 시간 -90분)
- **생체리듬 맵**: 24시간 내 최적 식사·운동·수면 타임라인을 개인 프로필 기반으로 동적 생성

### 2.3 Progressive Unlocking 시스템

사용자 지속률을 높이기 위한 주차별 루틴 단계적 해제 알고리즘:

| 주차 | 해제 루틴 | 근거 |
|------|----------|------|
| **Week 1** | 루틴 3·4·5 (식사 순서·식후 산책·수면 준비) | 기초 인크레틴 활성화 3종 |
| **Week 2** | 루틴 6·0 (14h 단식·아침 루틴) | 공복 의존성 형성 |
| **Week 3** | 루틴 1·2·7 + 위험 요인 8종 활성화 | 고급 루틴 + 리스크 인지 |
| **Week 4** | 루틴 8·9 (운동 강도·고급 단식) | 전체 시스템 완성 |

해제 조건: 전 주차 평균 IMEM 점수 ≥ 60점 또는 완료율 ≥ 70%.

### 2.4 AI 코치 봇 연동 시스템

- **텔레그램 챗봇**: grammY 프레임워크, 자연어 처리 → 루틴 체크·식사 기록·체중 기록
- **페르소나 3종**: 따뜻한 대사 코치(empathetic) / GLP-1 임상의(clinical) / 강인한 트레이너(driver)
- **Vision AI 식사 분석**: 사진 업로드 → 매크로 추정 → β_meal 자동 계산 (Anthropic Claude Vision API 활용)
- **프로액티브 코칭**: 13개 시간대 슬롯(06:30~22:00), 이벤트 기반 알림, 트렌드 분석(3일 연속 체중 증가·IMEM 하락·루틴 미스)

### 2.5 데이터 아키텍처

- **Firestore 실시간 동기화**: 앱(PWA) ↔ 텔레그램 봇 간 루틴·식사·체중 데이터 실시간 공유
- **Incremental Write 보호**: touched-set 기반 병합 저장 (봇과 앱의 쓰기 충돌 방지)
- **이벤트 로그 (Schema v2)**: append-only 이벤트 스트림 (감사 추적·임상 데이터 기반)

## 3. 종래 기술과의 차별점

| 구분 | 종래 기술 | 본 발명 (IMEM) |
|------|----------|--------------|
| **이론적 근거** | 칼로리 균형, 단순 습관 추적 | GLP-1 인크레틴 기전 기반 생리학적 모델 |
| **측정 방식** | 입력값의 합산·평균 | 계수 곱산 (비선형 상호작용 반영) |
| **식사 평가** | 칼로리만 계산 | 영양 순서 + 시간 + 매크로 구성 통합 평가 |
| **시간 개념** | 없음 또는 단순 타이머 | 24h 생체리듬 정렬 (골든타임, Bio-Sync) |
| **행동 유도** | 일괄 알림 | 계수별 실시간 피드백 + 단계적 루틴 해제 |
| **다채널 연동** | 앱 단독 | 앱 + AI 챗봇 + Vision AI 실시간 동기화 |
| **예측 모델** | 없음 | 4주 체중 변화 예측 (회귀 모델) |

**핵심 신규성 주장**: 인크레틴 호르몬 분비 최적화를 목적으로 일주기·영양·활동 3개 계수를 정의하고, 이를 곱산으로 결합하여 대사 효율을 정량화하는 알고리즘은 선행 기술에 존재하지 않습니다.

## 4. 청구 범위 후보 (초안)

변리사 검토를 위한 핵심 청구 후보 (아직 확정된 청구항이 아님):

**독립항 후보**
1. α(일주기)·β(영양)·γ(활동) 계수를 곱산으로 결합하여 인크레틴 대사 효율 점수를 산출하는 방법
2. β_meal 서브계수를 이용하여 식사 품질을 실시간으로 β 계수에 보정하는 방법
3. 위의 계수 산출 방법을 포함하는 생활습관 관리 시스템

**종속항 후보**
4. Progressive Unlocking 알고리즘을 이용한 루틴 단계적 해제 방법
5. 13슬롯 Bio-Sync 타이머 기반 프로액티브 알림 시스템
6. Vision AI를 이용한 식사 사진 분석 → β_meal 자동 산출 방법
7. 앱과 텔레그램 봇의 touched-set 기반 충돌 없는 데이터 동기화 방법

## 5. 실시 예증 (구현 현황)

본 발명은 현재 **실제 운영 중인 시스템**으로 구현되어 있습니다:

> **중요**: 일부 기능은 베타 사용자에게 노출된 상태이므로, 공개 기술 범위와 신규성 유지 기간(grace period)에 대해 변리사 검토 필요.

## 6. 변리사 검토 요청 사항

1. **신규성·진보성 조사**: α×β×γ 곱산 계수 방식 + β_meal 보정 알고리즘의 선행 기술 서치
2. **grace period 확인**: 베타 노출 시점 기준 국내·해외 출원 가능 기간 검토
3. **청구항 범위 설정**: 알고리즘 보호 범위 (방법/시스템/CRM 클레임) 구성 제안
4. **국제 출원 전략**: PCT 또는 미국/일본 우선 여부 (향후 GLP-1 디지털 치료제 시장 진입 고려)
5. **특허 #2 연계**: 동시 출원 또는 국내우선권(특허법 제55조) 주장 일정 협의
