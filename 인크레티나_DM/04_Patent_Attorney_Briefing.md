# Part 1. Patent #1 — IMEM (비만 적응증)

## 1.1 발명의 명칭 (후보)

변리사 검토를 통해 다음 중 선택 또는 변형:

| 후보 | 강조점 |
|------|------|
| **인크레틴 호르몬 자가분비 활성화를 위한 행동 패턴 정량화 시스템 및 그 방법** | 메커니즘 + 시스템 |
| **시간대별 행동 동기화 기반 대사 효율 산출 시스템** | Bio-Sync 강조 |
| **다중 계수 기반 인크레틴 효율 평가 및 코칭 시스템** | 알고리즘 강조 |

→ 변리사 권장: 한국 특허청은 "방법 + 시스템"을 함께 청구하는 것이 등록률 ↑.

## 1.2 기술 분야

- 디지털 헬스 (Digital Health)
- 의료 정보 시스템 (Medical Informatics)
- 행동 변화 기술 (Behavior Change Technology)
- 인공지능 기반 건강 관리 (AI-driven Health Management)

**IPC 분류 (예상)**:
- G16H 50/30 (건강관리 정보 처리 — 환자 데이터 분석)
- G16H 20/60 (식이 관리)
- G16H 20/30 (운동 관리)

## 1.3 배경기술 / 종래기술의 문제점

### 1.3.1 종래기술 카테고리 1 — 단순 칼로리 로깅 앱

| 대표 제품 | 한계 |
|---------|------|

| 대표 제품 | 한계 |

### 1.3.3 종래기술 카테고리 3 — 일반 헬스 코칭 앱

| 대표 제품 | 한계 |

### 1.3.4 종래 의약 — GLP-1 주사제

Ozempic, Wegovy, Mounjaro 등 GLP-1 작용제 약물:
- 외인성 GLP-1 주입 — 본 발명과 작용 결과는 같으나 메커니즘 정반대 (자가분비 활성화 vs 외부 주입)
- 부작용·비용·중단율 → 라이프스타일 대안 시장 확장

### 1.3.5 종합 — 미해결 과제

기존 어떤 디지털 헬스 솔루션도 다음을 동시에 충족하지 못함:

1. **인크레틴 호르몬 메커니즘**에 기반한 정량 모델
2. **시간(α) + 식이 시퀀스(β) + 활동(γ)** 3축 통합
3. **실시간 시간 정렬** (Bio-Sync) — 호르몬 골든타임 동기화
4. **자연어 AI 코치**와 정량 엔진의 통합
5. **위험 행동 → 즉각 회복 미션** 연동

## 1.4 해결하려는 과제

본 발명은 다음 과제를 해결한다:

1. 사용자의 일상 행동(식사·운동·수면 등)이 **인크레틴 호르몬 자가분비 효율**에 미치는 영향을 정량화
2. 행동의 **시간(언제), 순서(어떻게), 강도(얼마나)**를 단일 점수 체계로 통합
3. **실시간 골든타임 알림**으로 호르몬 분비 최적 시점에 행동 유도
4. 위험 행동 발생 시 **즉각 회복 미션** 자동 부여로 점수 회복
5. **자연어 AI 코치**가 점수·맥락을 이해하고 개인화 권고 제공

## 1.5 해결 수단 (핵심 발명 요소)

본 발명의 핵심 구성요소는 다음과 같다:

### 요소 ①: 다중 계수 정량 모델 — α × β × γ

```
대사 효율(Efficiency) = α_net × β_net × γ_net

α_net = α_base × ∏(α_modifiers) − Σ(α_penalty) + Σ(α_recovery)
β_net = β_base × β_meal − Σ(β_penalty) + Σ(β_recovery)
γ_net = γ_base × ∏(γ_modifiers) − Σ(γ_penalty) + Σ(γ_recovery)
```

- **α (일주기)**: 식사·수면·광노출 시간 정렬도
- **β (영양 시퀀스)**: 채소→단백질→탄수 순서 + β_meal 보정
- **γ (신체 활동)**: 식후 보행·근력 운동의 시점·강도

### 요소 ②: β_meal 식사 보정 알고리즘

```
β_meal = 0.95 + avgBetaScore × 0.075   (범위: 0.95 ~ 1.025)

avgBetaScore = (시퀀스 점수 + 단백질 비율 + 채소 비율) / 3
```

식사별 호르몬 분비 효율을 0.95~1.025 사이로 보정하여 β_net에 곱셈 반영.

### 요소 ③: Bio-Sync 시간 정렬 (골든타임 알고리즘)

24시간 중 호르몬 분비 최적 시점(골든타임)을 다음 마커로 정의·동기화:

| 마커 | 시간 | 임상 의미 |
|------|------|---------|
| 광노출 골든 | 기상 + 30분 내 | 코르티솔·멜라토닌 리셋 |
| 프리로드 윈도우 | 식사 30분 전 | 인크레틴 1차 분비 점화 |
| 식이 시퀀스 윈도우 | 식사 시작 후 5분 | β 직접 자극 |
| GLUT4 골든 | 식후 15-45분 | 글루코스 클리어런스 |
| 메타볼릭 스위치 | 19시 마감 | 야간 지방 저장 차단 |
| 멜라토닌 가드 | 22시 이후 | 수면 호르몬 보호 |

→ 각 마커에 카운트다운 알림·미션 연동.

### 요소 ④: 진보적 잠금 해제 (Progressive Unlocking)

신규 사용자에게 모든 루틴을 즉시 노출하면 부담 → 주차별 점진 해제:

```
Week 1: 루틴 [3, 4, 5] (식이 시퀀스·보행) — 가장 효과 큰 3개부터
Week 2: 루틴 [6, 0]    (저녁 마감·기상)
Week 3: 루틴 [1, 2, 7] (커피·단식 해제·근력) + 위험 카탈로그 활성화
Week 4: 루틴 [8, 9]    (멜라토닌·수면)
```

→ 행동 변화 심리학(Tiny Habits, BJ Fogg) 적용한 단계적 부담 증가.

### 요소 ⑤: 위험-회복 페어 시스템

8개 위험 행동 (R-01 ~ R-08) + 각 위험의 **즉각 회복 미션**:

| 위험 | 페널티 | 회복 미션 | 회복 점수 |
|------|------|---------|--------|
| R-04 식사 시퀀스 위반 | -10 | 채소 5분 먼저 씹기 | +6 |
| R-06 19시 이후 식사 | -12 | 내일 단식 1시간 연장 | +8 |
| R-07 야식 | -20 | 즉시 양치 + 20분 스트레칭 | +8 |

→ "위반 → 페널티 → 즉각 회복 → 점수 복원" 닫힌 루프 설계.

### 요소 ⑥: 자연어 AI 코치 통합

Claude API + 텔레그램 봇 통합:
- 사용자 자연어 입력 → IMEM 점수·컨텍스트 자동 호출
- 페르소나 3종 (따뜻한 코치 / 임상의 / 트레이너) — 사용자 선택
- 도구 호출 (tool use) — 식사 기록·체중 입력·루틴 체크 자연어 처리
- 시간대·페르소나·주차별 맥락 반영 메시지

### 요소 ⑦: 13슬롯 프로액티브 코칭 스케줄

```

11:00 preLunchCoaching   — 점심 프리코칭
11:30 lunchGolden        — 점심 골든타임
```

→ 사용자가 능동적으로 앱을 열지 않아도 시간대별 자동 코칭.

## 1.6 기술적 효과

본 발명이 달성하는 기술적 효과:

1. **정량화 가능한 행동 점수** — 모호한 "건강 습관"을 0-100 점수로 측정 가능
2. **호르몬 분비 메커니즘 기반** — 휴리스틱이 아닌 과학적 근거 (다수 RCT)
3. **시간 단위 정밀도** — 분 단위 골든타임 알림으로 행동 효과 극대화
4. **실시간 인과관계 가시화** — 행동 → 점수 변화 즉각 표시 → 동기 유지
5. **위험-회복 페어** — 위반 후에도 즉각 회복 가능 → 포기 방지
6. **자연어 인터페이스** — 입력 부담 최소 (텔레그램 채팅으로 모든 기록)
7. **확장 가능 아키텍처** — 캘리브레이션 분기로 다른 적응증으로 확장 (특허 #2 기반)

## 2.5 해결 수단 (핵심 발명 요소)

### 요소 ①: 6계수 통합 효율 공식

```
대사 효율(Efficiency_DM) = α_net × β_net × γ_net × δ_net × ε_net × ζ_net
```

기존 IMEM(Patent #1)의 α·β·γ에 다음 신규 계수 통합:

- **δ (Delta)**: 혈당 변동성 (Glycemic Variability) — TIR, GVI, MAGE, CV, AUC 통합 산출
- **ε (Epsilon)**: 약물 효과 보정 — 메트포민·GLP-1·인슐린 등 약물별 계수 영향 분리
- **ζ (Zeta)**: 스트레스·코르티솔 보정 — HRV·자기보고·수면 질 통합

### 요소 ②: δ_glycemic 산출 알고리즘

```
δ_glycemic = f(TIR, GVI, MAGE, CV, AUC_postprandial)

여기서:
TIR  = (70-180 mg/dL 시간 / 총 시간) × 100  (%)
GVI  = 곡선 길이 / 직선 길이  (변동성)
MAGE = 평균 변동 폭 (mg/dL)
CV   = 표준편차 / 평균 × 100  (%)
AUC_postprandial = 식후 곡선 아래 면적 (mg/dL × min)

δ_net = (TIR/100) × (1 - CV/100) × variability_modifier
```

→ 국제 당뇨 학회(ADA, EASD) 표준 지표 기반.

### 요소 ③: ε_med 약물 효과 분리 알고리즘

```
ε_net = ∏(약물별 modifier) × 복약 순응도

약물별 modifier 예시:
메트포민 (1000mg/일):    β·γ × 1.10
GLP-1 주사 (semaglutide): α·β × 1.20
DPP-4 억제제:            β × 1.05
SGLT-2 억제제:           δ × 1.10
인슐린 (속효성):         β_meal 효과 분리 (행동 점수에서 약 효과 차감)
인슐린 (지속형):         α 공복혈당 부분 분리

복약 순응도 = (실제 복용 / 예정 복용) × adherence_weight
```

**사용자 표시 정책 — 핵심 발명**:
```
오늘의 IMEM 점수: 78점
├─ 행동 점수: 62점 (당신의 노력)
└─ 약물 효과: +16점 (메트포민 + GLP-1 주사)

★ 약 의존도: 21% (지난 달 대비 -3%)
```

→ 환자가 "약 줄이기" 목표를 정량적으로 추적 가능.

### 요소 ④: 3단계 모드 시스템 (Basic / Standard / Full)

데이터 가용성에 따라 자동 전환:

| 모드 | 활성화 조건 | 사용 계수 | 효율 정확도 |
|------|----------|---------|---------|
| **Basic** | 식사·운동·수면만 | α, β, γ | 60% |
| **Standard** | + 자가 혈당 측정 (1일 1-2회) | + δ (부분), + ε | 80% |
| **Full** | + CGM + 웨어러블 | 전체 6계수 | 95% |

→ 사용자가 CGM 없이 시작해도 가치 받고, 점진 정밀화.

### 요소 ⑥: 안전 레이어 (Safety Filter)

모든 IMEM-DM 권고는 안전 필터 통과 후에만 노출:

```
권고 생성 → Safety Filter
              │
              ├─ 저혈당 위험 체크 (인슐린·SU 사용자 + 공복혈당 < 100)
              ├─ 약물 상호작용 체크
              ├─ 식이 제한 체크
              └─ 운동 적합성 체크 (혈당 > 250 → 운동 차단)
                              │
                              ▼
                      통과: 권고 노출
                      차단: 대안 권고
```

**6가지 안전 규칙** (예시):
- 인슐린 사용자 + 공복혈당 < 100 → 운동 권고 차단, 10g 탄수 권유
- 혈당 > 250 + 케톤 양성 → 운동 연기 권유, 의료진 상담
- 인슐린 사용자에게 14h+ 단식 권고 차단

### 요소 ⑦: PPG 사후 보정 알고리즘

```

실측 PPG = 사용자 입력 또는 CGM 자동 측정
오차 = 실측 PPG - 예측 PPG

if (|오차| > threshold):
  → β_meal 캘리브레이션 자동 조정
  → 사용자에게 원인 분석 (메뉴 추정 오류, 약물 변동, 스트레스 등)
```

- IMEM 점수 추세
- 복약 순응도
- 환자 자기보고 컨디션

→ **종래기술과의 결정적 차별점**: 행동 점수와 임상 지표를 통합한 단일 보고서.

### 요소 ⑨: 신규 위험 카탈로그 (R-09 ~ R-15)

당뇨 전용 7개 신규 위험:
- R-09 새벽현상
- R-10 저혈당
- R-11 약물 누락
- R-12 스트레스성 고혈당
- R-13 운동 후 저혈당 (인슐린)
- R-14 식후 PPG 스파이크
- R-15 야간 혈당 변동

기존 R-01 ~ R-08도 페널티 재조정 (R-04: -10 → -18, R-06: -12 → -20 등).

### 요소 ⑩: Calibration Profile 시스템

단일 코어 엔진이 다중 적응증 캘리브레이션을 지원:

```
calculateIMEM({
  ...standardArgs,
  calibrationProfile: 'obesity' | 'diabetes' | 'prediabetes',
  cgmData: { ... } | null,
  medications: [ ... ] | null,
})
```

→ 향후 GDM(임신성 당뇨), NAFLD(비알코올성 지방간) 등 추가 적응증으로 확장 가능 (확장성 청구).

## 2.6 기술적 효과

본 발명(Patent #2)이 달성하는 기술적 효과:

1. **혈당 변동성 정량화** — 평균 HbA1c만으로 부족한 합병증 위험을 다각도 평가
2. **약물 효과 분리** — 환자가 "행동의 순수한 가치"를 인지 → 약 줄이기 동기 강화
3. **CGM 단계적 활용** — 진입 장벽 낮추고 점진적 정밀화 (Basic → Full)
4. **24h 정렬** — 새벽현상부터 야간 회복까지 전 구간 행동 가이드
5. **인슐린 사용자 안전 보장** — 저혈당 위험 행동 사전 차단
6. **PPG 학습** — 사용자별 식이 반응 학습 → 개인화 정확도 향상
7. **임상 채널 진입** — 의사 보고서로 의료진 신뢰 확보 → 처방형 DTx 진출 경로
8. **확장 가능 아키텍처** — 단일 코어, 다중 적응증으로 사업 자산 극대화
