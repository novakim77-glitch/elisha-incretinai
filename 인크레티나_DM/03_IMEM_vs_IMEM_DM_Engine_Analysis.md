## 1. 분기의 본질 — 왜 새 엔진이 필요한가

### 1.1 GLP-1의 두 갈래 작용 (재확인)

GLP-1은 분비된 후 down-stream에서 두 개의 독립적 축으로 작용합니다:

```
                    GLP-1 자가분비 (IMEM 결과물)
                              │
            ┌─────────────────┴─────────────────┐
            ▼                                   ▼
    ① 췌장-혈당 축 (Pancreatic-Glycemic)       ② 뇌-식욕 축 (Neuro-Appetite)
    • 인슐린 분비 ↑ (글루코스 의존성)          • 시상하부 작용 → 식욕 ↓
    • 글루카곤 분비 ↓                          • 보상 회로 둔화
    • β세포 보호                              • 위 배출 지연
    • 인슐린 감수성 회복 (간접)               • 포만감 강화
            │                                   │
            ▼                                   ▼
       【당뇨 적응증】                      【비만 적응증】
       임상 지표: HbA1c, TIR, GMI          임상 지표: 체중, 체지방률
       병태생리: β세포 기능 + 인슐린 저항  병태생리: 에너지 균형 + 식욕 조절
```

### 1.2 IMEM(현재)이 비만에 최적화된 측면

현재 IMEM 엔진은 **암묵적으로 비만 적응증에 최적화**되어 있습니다:

| IMEM 요소 | 비만 최적화 가정 |
|----------|---------------|
| **출력 지표** | IMEM 점수 → 체중 예측 (4주 후) |
| **β_meal 범위** | 0.95~1.025 — 포만감 효율을 좁은 범위로 표현 |
| **R-06 (19시 이후 식사)** | -12점 — "지방 저장" 메타포로 페널티 |
| **루틴 카탈로그** | 14h 단식, 메타볼릭 스위치 등 비만 중심 |
| **AI 코치 페르소나** | 따뜻한 코치, 트레이너 — 동기부여 중심 |
| **Bio-Sync** | 14h 공복 → 지방 연소 |

### 1.3 IMEM-DM이 채워야 할 임상 갭

당뇨 적응증을 위해 다음 7가지 갭이 채워져야 합니다:

| # | 갭 | 의학적 중요성 |
|---|---|------------|
| 1 | **혈당 변동성 정량화 부재** | TIR·GVI·MAGE 같은 당뇨 표준 지표 누락 |
| 2 | **약물 효과 미반영** | 모든 당뇨 환자 약 복용 — 약효 보정 없으면 IMEM 점수 왜곡 |
| 3 | **저혈당 안전성 로직 부재** | 인슐린·SU 사용자 운동·단식이 위험할 수 있음 |
| 4 | **새벽현상 모니터링 부재** | 공복혈당 변동의 핵심 원인 |
| 5 | **CGM 데이터 통합 부재** | 24h 곡선 정보가 혈당 관리의 본질 |
| 6 | **스트레스성 고혈당 미반영** | 당뇨 환자의 주요 변동 요인 |
| 7 | **합병증 모니터링 부재** | 발·눈·신장 — 평생 관리 필수 |

→ **IMEM-DM은 위 7가지 갭을 체계적으로 채우는 확장 엔진**.

### 1.4 핵심 설계 원칙

IMEM-DM 설계는 다음 5가지 원칙을 따릅니다:

1. **하위 호환성** — IMEM의 α/β/γ 계수는 그대로 유지하되 캘리브레이션만 분기
2. **점진적 활성화** — CGM 없는 사용자도 사용 가능 (degraded mode), CGM 연동 시 풀 활성화
3. **안전성 우선** — 모든 신규 권고는 저혈당·약물 상호작용 안전 체크 통과 후 노출
4. **임상 표준 정합성** — TIR·HbA1c 등 국제 당뇨 학회(ADA, EASD) 표준 지표 채택
5. **확장 가능성** — 향후 전당뇨, GDM(임신성 당뇨) 등 추가 적응증 확장 여지 보존

## 2. 핵심 공식의 진화

### 2.1 기존 IMEM 공식

```
대사 효율 (Efficiency) = α_net × β_net × γ_net

α_net = α_base × ∏(α_modifiers) − Σ(α_penalty) + Σ(α_recovery)
β_net = β_base × β_meal − Σ(β_penalty) + Σ(β_recovery)
γ_net = γ_base × ∏(γ_modifiers) − Σ(γ_penalty) + Σ(γ_recovery)
```

여기서:
- `*_base`: 해당 시점 기본 계수 (시간대별 변화)
- `*_modifiers`: 행동·조건 기반 곱셈 보정 (예: β_meal)
- `*_penalty`: 위험 행동 시 차감
- `*_recovery`: 회복 미션 완료 시 가산

### 2.2 IMEM-DM 확장 공식

```
대사 효율 (Efficiency_DM) = α_net × β_net × γ_net × δ_net × ε_net × ζ_net

여기서:
α_net, β_net, γ_net = 동일 골격, 다른 캘리브레이션
δ_net = Glycemic Variability 계수 (CGM 활성 시)
ε_net = 약물·복약 보정 계수
ζ_net = 스트레스·코르티솔 보정 계수
```

### 2.3 단계적 활성화 모드

사용자 데이터 가용성에 따라 IMEM-DM은 3가지 모드로 작동:

| 모드 | 활성화 조건 | 사용 계수 | 정확도 |
|------|----------|---------|------|
| **Basic Mode** | 식사·운동·수면만 입력 | α, β, γ (당뇨 캘리브레이션) | 60% |
| **Standard Mode** | + 자가 혈당 측정 (1일 1-2회) | + δ (부분), + ε (복약) | 80% |
| **Full Mode** | + CGM + 웨어러블 | 전체 6개 계수 | 95% |

→ 사용자가 CGM 없이 시작하더라도 가치를 받고, 점진적으로 정밀도가 올라감.

### 2.4 캘리브레이션 분기

같은 α/β/γ 골격이라도 적응증별 가중치·범위가 다름:

```
calibration_obesity = {
  alpha:   { weight: 1.0, base: 1.0 },
  beta:    { weight: 1.0, base: 1.0, meal_range: [0.95, 1.025] },
  gamma:   { weight: 1.0, base: 1.0 },
  // 신규 계수 비활성
  delta: null, epsilon: null, zeta: null,
}

calibration_diabetes = {
  alpha:   { weight: 1.2, base: 1.0 },        // 일주기 가중치 ↑
  beta:    { weight: 1.0, base: 1.0, meal_range: [0.85, 1.05] }, // 범위 확장
  gamma:   { weight: 1.5, base: 1.0 },        // 활동 가중치 ↑
  delta:   { weight: 1.3, base: 1.0, mode: 'cgm_aware' }, // 신규
  epsilon: { weight: 1.0, base: 1.0, mode: 'med_aware' }, // 신규
  zeta:    { weight: 0.8, base: 1.0, mode: 'optional' },  // 신규
}
```

(위 수치는 설계 의도 표현용 예시. 최종 값은 임상 데이터 검증 후 확정.)

## 3. α (일주기) 계수 — 변화

### 3.1 의미의 확장

| 차원 | IMEM (비만) | **IMEM-DM (당뇨)** |
|------|------------|-------------------|
| **핵심 의미** | 대사 리듬 동조 → 지방 연소 효율 | **새벽현상 억제 + 인슐린 일중 효율 + 코르티솔 리듬** |
| **시간 단위** | 큰 시간 블록 (식사·수면·운동) | + 시간 단위 정밀화 (식후 1h, 2h, 새벽 4-8시) |
| **임상 가중치** | 1.0 | **1.2** (당뇨에서 일주기 깨짐이 핵심 병태생리) |

### 3.2 신규 입력 데이터

기존 IMEM α 입력:
- 기상 시간
- 광노출 시간
- 식사 시간 패턴
- 수면 시간

IMEM-DM α 추가 입력:
- **새벽 코르티솔 패턴** (자가 보고 또는 웨어러블 추정)
- **수면 분절 빈도** (sleep fragmentation)
- **수면 단계** (REM·NREM 비율, 웨어러블)
- **공복 시간 정밀화** (last calorie → next calorie, 분 단위)

### 3.3 α 계수 산출 로직 변화

```
[기존 IMEM α]
α_modifier_meal_timing = 식사 시간이 일주기와 정렬되었는지 (이진 또는 단순 점수)

[IMEM-DM α 신규 보정]
α_modifier_dawn = 새벽현상 위험도 (전날 저녁 식사 → 새벽 코르티솔 → 공복혈당 예측)
α_modifier_sleep_quality = 수면 분절 + REM 비율 → 인슐린 감수성 영향
α_modifier_fasting_duration = 공복 시간 vs 다음 식사 인슐린 반응 예측
```

### 3.4 R-06 (19시 이후 식사) 페널티 재조정

비만에서 R-06은 "지방 저장" 메타포로 -12점.
당뇨에서 R-06은 **이중 메커니즘**으로 작동:
1. 야간 인슐린 저항성 증가 (직접 PPG 상승)
2. 코르티솔 분비 교란 → 새벽현상 유발

→ **IMEM-DM에서 R-06 페널티: -20점**

### 3.5 신규 α 위험 (R-09 신규)

**R-09: 새벽현상 (Dawn Phenomenon)**
- 트리거: 공복혈당 130 mg/dL 이상 또는 전날 저녁-당일 새벽 혈당 상승 패턴
- 페널티: -8점 (α 계수에 직접 적용)
- 회복 미션:
  - 전날 19시 이전 마지막 식사 완료
  - 전날 30분 저녁 산책
  - 당일 기상 직후 광노출 10분

## 4. β (영양 시퀀스) 계수 — 변화

### 4.1 의미의 확장

| 차원 | IMEM (비만) | **IMEM-DM (당뇨)** |
|------|------------|-------------------|
| **핵심 의미** | 인크레틴 분비 효율 → 포만감 → 칼로리 적자 | **식후 혈당 스파이크 평탄화 (PPG)** + 인크레틴 분비 |
| **β_meal 범위** | 0.95~1.025 (좁은 범위) | **0.85~1.05 (넓은 범위)** |
| **임상 가중치** | 1.0 | 1.0 (동일, 단 범위가 넓음) |

### 4.2 β_meal → β_glycemic 진화

기존 β_meal 공식:
```
β_meal = 0.95 + avgBetaScore × 0.075
avgBetaScore = (시퀀스 점수 + 단백질 비율 + 채소 비율) / 3
```

IMEM-DM β_glycemic 신규 공식 (개념):
```
β_glycemic = 0.85 + (avgBetaScore × 0.10) + (GI_modifier × 0.05) + (fiber_modifier × 0.05)

여기서:
GI_modifier = (100 - 식사 평균 GI) / 100   // 저GI일수록 높음
fiber_modifier = min(식이섬유 g / 10, 1)    // 10g 이상이면 1.0
```

### 4.3 신규 β 입력 데이터

기존 IMEM β 입력:
- 식사 시간
- 메뉴 (텍스트)
- 시퀀스 (채소→단백질→탄수)
- 칼로리

IMEM-DM β 추가 입력:
- **GI (Glycemic Index)** — 메뉴 자동 매핑 또는 사진 분석
- **GL (Glycemic Load)** = GI × 탄수 g / 100
- **식이섬유 g** — 사용자 입력 또는 추정
- **단순당 g** (separately tracked)
- **단백질 절대량 g** (BMI 기반 목표 대비)

### 4.4 식후 PPG 측정 통합 (Standard/Full Mode)

자가 혈당 측정 또는 CGM 사용 시:
- **식후 1h PPG** 입력 → β_meal 사후 보정
- **식후 2h PPG** 입력 → β_meal 사후 보정
- 기록된 PPG가 예측치와 차이 클 시 → AI 코치가 원인 분석 (메뉴 추정 오류, 약 효과 변동 등)

### 4.5 R-04 (식이 시퀀스 위반) 페널티 재조정

비만에서 R-04: -10점
당뇨에서 R-04는 **즉각적 PPG 스파이크 직격** → -18점

회복 미션 강화:
- 비만: 채소 5분 먼저 씹기
- 당뇨: 채소 5분 먼저 + 식후 즉시 10분 보행 (PPG 클리어런스)

### 4.6 신규 β 위험 (R-14 신규)

**R-14: 식후 혈당 스파이크 (PPG ≥ 200 mg/dL)**
- 트리거: 측정된 식후 1-2h 혈당 200 이상
- 페널티: -10점
- 회복 미션:
  - 즉시 15분 빠른 보행 (GLUT4 동원)
  - 다음 끼니 채소 분량 두 배
  - 다음 끼니 단순당 차단

## 5. γ (신체 활동) 계수 — 변화

### 5.1 의미의 확장

| 차원 | IMEM (비만) | **IMEM-DM (당뇨)** |
|------|------------|-------------------|
| **핵심 의미** | 근손실 방지 → 대사율 유지 | **인슐린 감수성 직접 회복 + GLUT4 동원** |
| **임상 가중치** | 1.0 | **1.5** (당뇨에서 운동 효과 즉각적·결정적) |
| **시간 정밀도** | 식후 30-60분 | **식후 15-90분 (정밀 GLUT4 윈도우)** |

### 5.2 식후 운동 시점의 정밀화

비만에서 식후 보행은 "글루코스 클리어런스" 일반론.
당뇨에서는 시점·강도가 인슐린 반응에 직접 영향:

| 식후 시점 | 운동 효과 |
|---------|---------|
| 0-15분 | 소화 방해 (피해야 함) |
| **15-45분** | **GLUT4 최대 동원, PPG 피크 억제 (최적)** |
| 45-90분 | 후기 PPG 감소 |
| 90분+ | 일반 칼로리 소모 |

→ IMEM-DM은 **15-45분 윈도우를 골든 윈도우**로 카운트다운.

### 5.3 운동 강도 vs 인슐린 반응

기존 IMEM γ는 강도를 단순 분류 (저/중/고).
IMEM-DM은 강도-인슐린 반응 매핑:

| 강도 | 인슐린 감수성 영향 | 저혈당 위험 |
|------|----------------|-----------|
| 저 (걷기) | + (간접) | 없음 |
| 중 (조깅, 자전거) | ++ | 약물 사용 시 주의 |
| 고 (HIIT, 무거운 근력) | +++ (즉각·24h 지속) | **인슐린 사용자 위험 ↑** |

### 5.4 운동 후 혈당 측정 통합

CGM 또는 자가측정 사용 시:
- 운동 직후 혈당 → γ 효과 검증
- 운동 후 2h 혈당 → 후기 효과 검증
- 운동 후 야간 혈당 → 야간 저혈당 모니터링 (인슐린 사용자)

### 5.5 신규 γ 위험 (R-13 신규)

**R-13: 운동 후 저혈당 (인슐린·SU 사용자)**
- 트리거: 운동 후 혈당 < 70 mg/dL
- 페널티: -8점
- 회복 미션:
  - 즉시 15g 단순당 (사탕, 주스)
  - 15분 후 재측정
  - 다음 운동 전 10g 탄수 사전 섭취

### 5.6 인슐린 사용자 운동 가이드 (안전 레이어)

IMEM-DM은 **인슐린·SU 사용자에게 운동 권유 시 사전 가이드**:
- 운동 전 혈당 < 100 → 10g 탄수 먼저
- 운동 전 혈당 100-180 → 정상 진행
- 운동 전 혈당 > 250 → 운동 연기 (케토산증 위험)
- 운동 후 30분·2h 측정 권고

## 6. δ (Glycemic Variability) 계수 — 신규

### 6.1 도입 배경

당뇨 관리의 본질은 **혈당 평균(HbA1c)뿐 아니라 변동성 (variability)**입니다. 같은 HbA1c 7%여도:
- A 환자: 안정적 변동, TIR 80% — 양호
- B 환자: 큰 변동, TIR 50% — 합병증 위험 ↑

→ **δ_glycemic 계수**로 변동성을 정량화.

### 6.2 δ 산출 구성요소

```
δ_glycemic = f(TIR, GVI, MAGE, CV, AUC_postprandial)

여기서:
TIR  (Time in Range): 70-180 mg/dL 시간 비율 (목표 ≥ 70%)
GVI  (Glucose Variability Index): 곡선 길이 vs 직선 비율
MAGE (Mean Amplitude of Glycemic Excursions): 평균 변동 폭
CV   (Coefficient of Variation): 표준편차 / 평균 (목표 < 36%)
AUC_postprandial: 식후 곡선 아래 면적
```

### 6.3 δ 계산 정책

**Full Mode (CGM 활성)**:
- δ = 실시간 산출 (5분 단위 CGM 데이터)
- 일일 δ_net = (TIR/100) × (1 - CV/100) × variability_modifier

**Standard Mode (자가측정)**:
- δ = 부분 추정 (1일 3-5회 측정 → 패턴 보간)
- 정확도 60-70%로 표시

**Basic Mode (혈당 데이터 없음)**:
- δ = 1.0 (중립, 영향 없음)
- 대신 행동 기반 추정 표시 (β·γ 점수로 간접 추정)

### 6.4 δ 가중치 정책

```
효율 기여도:
Full Mode:      α(20%) × β(25%) × γ(25%) × δ(30%)
Standard Mode:  α(25%) × β(30%) × γ(25%) × δ(20%)
Basic Mode:     α(30%) × β(35%) × γ(35%) × δ(0%)
```

→ CGM 사용자일수록 δ가 IMEM 점수에서 큰 비중.

### 6.5 δ 임상 가치

δ가 정량화하는 지표는 **국제 당뇨 학회(ADA, EASD) 표준**:
- TIR ≥ 70% → 합병증 위험 유의 감소 (Battelino et al., Diabetes Care 2019)
- CV < 36% → "Stable" 분류
- GVI < 1.4 → "Optimal"

→ **임상 학회와 정합되는 지표 채택으로 의료진·보험사 신뢰성 확보**.

## 7. ε (약물·복약 보정) 계수 — 신규

### 7.1 도입 배경

당뇨 환자의 95% 이상이 약 복용 중. 약효를 IMEM에 반영하지 않으면:
- 약 덕분에 좋아진 혈당을 "행동 효과"로 오인 → 사용자 혼란
- 약 효과 변동 (주사 시점, 복약 누락) 미반영 → 점수 왜곡

→ **ε_med 계수**로 약물 효과를 IMEM 점수에서 정확히 분리.

### 7.2 약물 카테고리 매핑

| 약물 종류 | 작용 메커니즘 | IMEM 계수 영향 |
|---------|----------|--------------|
| **메트포민** | 간 포도당 생성 ↓, 인슐린 감수성 ↑ | β·γ 계수 증폭 (×1.1) |
| **GLP-1 주사** (Ozempic 등) | 외인성 GLP-1 | α·β 계수 증폭 (×1.2) — 자가분비 + 외부 합산 |
| **DPP-4 억제제** | 내인성 GLP-1 분해 차단 | β 계수 증폭 (×1.05) |
| **SGLT-2 억제제** | 신장 포도당 재흡수 차단 | δ 계수 증폭 (×1.1) — 변동성 감소 |
| **인슐린 (속효성)** | 식후 혈당 직접 강하 | β_meal 효과를 "약 효과"로 분리 |
| **인슐린 (지속형)** | 기저 혈당 강하 | α 계수의 "공복혈당" 부분을 약 효과로 분리 |
| **SU (설포닐우레아)** | 인슐린 분비 강제 자극 | β 계수 분리 + R-13 위험 ↑ |

### 7.3 ε 산출 로직

```
ε_net = ∏(약물별 modifier) × 복약 순응도

복약 순응도 = (복용 완료 / 복용 예정) × adherence_weight
```

**복약 순응도** 입력:
- 약 복용 시간 자가 입력
- 알림 응답률
- 향후 스마트 약통 연동 가능

### 7.4 ε의 표시 정책 — 사용자 투명성

IMEM 점수 표시 시 ε 효과를 **숨기지 않고 분리 표시**:

```
오늘의 IMEM 점수: 78점
├─ 행동 점수: 62점 (당신의 노력)
└─ 약물 효과: +16점 (메트포민 + GLP-1 주사)

★ 약 의존도: 21% — 지난 달 대비 -3% (행동 비중 증가!)
```

→ **약을 줄이는 것이 목표라면 행동 점수 증가가 핵심**임을 사용자에게 명확히.

### 7.5 ε 위험 (R-11 신규)

**R-11: 약물 복용 누락**
- 트리거: 예정 복용 시간 후 30분 미체크
- 페널티: -10점 (ε 계수에 직접 적용)
- 회복 미션:
  - 다음 식사 전 즉시 복용 (GLP-1·메트포민)
  - 인슐린의 경우: 의료진 지침에 따른 보정
  - 알림 강화 설정

## 8. ζ (스트레스·코르티솔) 계수 — 신규

### 8.1 도입 배경

스트레스성 고혈당은 당뇨 환자의 주요 변동 요인:
- 코르티솔·아드레날린 → 간 포도당 방출 → 혈당 ↑
- 만성 스트레스 → 인슐린 저항성 ↑
- 수면 부족 → 코르티솔 ↑ → 새벽현상 악화

→ **ζ_stress 계수**로 스트레스 영향을 보정.

### 8.2 ζ 입력 데이터

| 입력 종류 | 출처 | 가용성 |
|---------|------|------|
| **HRV (심박변이도)** | 웨어러블 (Apple Watch, Garmin, Fitbit) | Standard/Full Mode |
| **자기보고 스트레스 점수** (1-10) | 앱 일일 입력 | 모든 Mode |
| **수면 질** | 웨어러블 또는 자가보고 | 모든 Mode |
| **혈당 변동 패턴** | CGM 분석 → 스트레스성 변동 추정 | Full Mode |

### 8.3 ζ 산출 로직 (개념)

```
ζ_stress = baseline_resilience × ∏(stress_modifiers)

stress_modifiers:
- hrv_modifier:    HRV 7일 trend (감소 → 스트레스 ↑)
- sleep_modifier:  수면 질 (분절·시간)
- self_report:     자기보고 점수
```

### 8.4 ζ의 표시 정책

ζ는 **권고가 아닌 인지 도구**로 활용:
- 스트레스 ↑ 감지 → "혈당 변동성이 클 수 있어요. 평소보다 한 번 더 측정해보세요"
- 수면 부족 ↑ → "오늘 새벽현상 위험 ↑. 저녁 마감 시간 더 일찍"

→ **ζ는 자동 점수 페널티가 아닌 컨텍스트 보정**.

### 8.5 ζ 위험 (R-12 신규)

**R-12: 스트레스성 고혈당 의심**
- 트리거: 정상 식이·운동에도 불구하고 혈당 200 이상 (식사 무관)
- 페널티: -6점 (ζ 계수에 적용)
- 회복 미션:
  - 5분 호흡 운동 (4-7-8 호흡법)
  - 미주신경 활성화 (찬물 세안, 콧노래)
  - 30분 후 혈당 재측정

### 9.1 데이터 채널 비교 매트릭스

| **자기보고** | (없음) | **스트레스 점수, 컨디션 1-10** |

### 9.2 CGM 통합 — 핵심 신규 채널

CGM(연속혈당측정) 통합은 IMEM-DM의 **결정적 기술 자산**:

**지원 디바이스**:
- Dexcom G7 (글로벌 표준)
- Freestyle Libre 3 (한국 보급률 ↑)
- Medtronic Guardian
- 한국 디바이스 (필요시 SDK 협업)

**데이터 처리 파이프라인**:
```
CGM 디바이스 → BLE 연동 → 5분 단위 혈당값
   ↓
앱 로컬 저장 (배터리 절약)
   ↓
주기적 동기화 (1시간 단위) → Firestore
   ↓
실시간 계산 엔진 → δ, AGP, TIR 산출
   ↓
사용자 알림 (저혈당, 고혈당, 트렌드 변화)
```

### 9.3 데이터 입력 부담 최소화 전략

당뇨 환자는 이미 **로깅 부담**을 느끼는 경우 多 → IMEM-DM은 입력 자동화 우선:

| 항목 | 자동화 방식 |
|------|----------|
| 혈당 | CGM 자동 (Full Mode) |
| 식사 | 사진 분석 (Claude Vision) + GI 자동 매핑 |
| 운동 | 웨어러블 자동 동기화 (Apple Health, Google Fit) |
| 복약 | 알림 응답 (탭 한 번) |
| HbA1c | 분기 1회 수동 입력 (또는 병원 EMR 연동 — 장기) |

당뇨 환자는 정기적으로 의사 진료. IMEM-DM은 **의사 진료에 활용 가능한 보고서**를 자동 생성:

**의사 보고서 구성** (월 1회):
1. AGP 곡선 (지난 14일)
2. TIR / 평균 혈당 / GMI
3. 저혈당·고혈당 에피소드 빈도
4. 식이·운동 패턴 요약
5. 복약 순응도
6. IMEM 점수 추세
7. 환자 자기보고 컨디션

→ **의료진이 진료 시 활용 → 의료진 추천 → 신뢰 채널 형성**.

| ID | 위험 | 페널티 | 계수 | 회복 미션 |
|----|------|------|------|----------|
| **R-09** | 새벽현상 (공복혈당 130+) | -8 | α | 전날 19시 마감 + 저녁 산책 |
| **R-10** | 저혈당 (<70 mg/dL) | -15 | (안전) | 즉시 15g 단순당 + 의료진 보고 |
| **R-11** | 약물 복용 누락 | -10 | ε | 다음 식사 전 즉시 복용 |
| **R-12** | 스트레스성 고혈당 | -6 | ζ | 5분 호흡 + 미주신경 활성화 |
| **R-13** | 운동 후 저혈당 (인슐린) | -8 | γ | 운동 전 10g 탄수 (다음번) |
| **R-14** | 식후 200+ 스파이크 | -10 | β, δ | 즉시 15분 보행 + 다음끼니 채소 ×2 |
| **R-15** | 야간 혈당 변동 | -8 | α, δ | 저녁 마감 시간 조정 |

### 12.4 위험 페널티 적용 정책

**저혈당 (R-10)의 특수성**:
- 저혈당은 "행동 위험"이 아닌 "안전 위험" → 페널티는 환자에게 표시하지 않고 의료진 보고용으로만 기록
- 환자에게는 **회복 미션과 안전 정보**만 표시
- 의료진 보고서에 빈도·시점 자동 기록

→ "건강 점수 -15점"은 환자에게 부정적. 대신 "저혈당 발생 — 안전 우선 조치를 안내합니다".

### 12.5 위험-회복 연동 강화

비만에서 회복 미션은 +3~+8점.
당뇨에서는 일부 회복 미션이 **즉시 측정 검증**으로 완료됨:

예: R-14 (식후 스파이크) → 회복 미션 "15분 보행" → 30분 후 재측정 시 혈당 30 mg/dL 이상 감소 → +12점 (검증 완료 보너스)

→ **행동-결과의 즉각적 인과관계**를 사용자에게 보여줌. 이는 비만에서 4주 후 체중 변화로만 검증 가능했던 것과 차별점.

## 14. 안전성 레이어 (Safety Layer) — 신규

### 14.1 안전성의 절대 우위

당뇨 환자(특히 인슐린·SU 사용자)는 **저혈당이 즉각적 생명 위험**. IMEM-DM의 모든 권고는 안전 체크 통과 후에만 노출:

```
IMEM 권고 생성
    │
    ▼
┌─────────────────────┐
│  Safety Filter      │
│  ─────────────────  │
│  • 저혈당 위험 체크 │
│  • 약물 상호작용    │
│  • 식이 제한 체크   │
│  • 운동 적합성      │
└─────────────────────┘
    │
    ▼
권고 노출 (필터 통과 시만)
또는
대안 권고 (필터 차단 시)
```

### 14.2 안전 규칙 카탈로그

| 안전 규칙 | 대상 | 차단 권고 | 대안 |
|---------|------|--------|------|
| **공복 운동 차단** | 인슐린·SU 사용자 + 공복혈당 < 100 | "운동 전 식이 권유" | 10g 탄수 후 운동 |
| **고강도 운동 차단** | 혈당 > 250 또는 케톤 양성 | "운동 연기 권유" | 의료진 상담 |
| **단식 권고 차단** | 인슐린 사용자 | 14h+ 단식 권고 안 함 | 12h 가벼운 단식만 |
| **저혈당 후 운동 차단** | 최근 24h 저혈당 발생 | 운동 권고 보류 | 의료진 확인 |
| **약물-식이 상호작용** | 메트포민 + 알코올 | 알코올 경고 | 식사 직후 권유 |
| **알코올 + 인슐린** | 인슐린 사용자 + 알코올 입력 | 야간 저혈당 경고 | 취침 전 측정 |

### 14.3 응급 알림 시스템

CGM Full Mode에서:
- 혈당 < 70 → 저혈당 알림 (앱 + 푸시)
- 혈당 < 55 → **긴급 알림** (전화 알람 톤, 보호자 자동 알림 옵션)
- 혈당 > 300 → 고혈당 알림 (의료진 상담 권유)
- 혈당 > 400 → 긴급 알림 (응급실 권유)

### 14.4 의료진 연동 채널

| 시나리오 | 자동 동작 |
|---------|--------|
| 1주 내 저혈당 3회 이상 | 의료진 보고서에 강조 표시 |
| 1주 내 혈당 > 300 5회 이상 | 사용자에게 의료진 상담 권유 + 보고서 알림 |
| HbA1c > 9.0 입력 시 | 의료진 즉시 진료 권유 |
| 케톤 양성 의심 (혈당 > 250 + 메스꺼움 자가보고) | 응급실 방문 권유 |

### 14.5 보호자 모드 (Optional)

가족·보호자가 환자 데이터 일부 공유 받을 수 있음:
- 저혈당 발생 시 자동 SMS
- 주간 혈당 트렌드 요약
- 약 복용 알림 누락 시 알림

→ 노년 환자(페르소나 C)에게 핵심 기능.

| 페르소나 | 톤 | 주요 사용 시점 |
|---------|-----|------------|
| **당뇨 전문 내과의** | 임상·정량적 | HbA1c·TIR 해석, 약 조정 논의 |
| **당뇨 교육 간호사** | 따뜻·교육적 | 신규 진단 사용자 onboarding |

### 15.3 페르소나 자동 추천 로직

사용자 프로필에 따라 기본 페르소나 자동 추천:

| 사용자 상태 | 추천 페르소나 |
|----------|-----------|
| 신규 진단 (1년 이내) | 당뇨 교육 간호사 |
| 약 복용 5년+ | 혈당 코치 |
| GLP-1 주사 사용 | GLP-1 약물 동반자 |
| 의료진 친화 / HbA1c 7+ | 당뇨 전문 내과의 |

→ 사용자가 언제든 변경 가능. 시간대별 자동 전환 옵션 (예: 아침은 따뜻한 톤, 저녁은 임상 톤).

### 15.4 페르소나별 메시지 차별화 예시

**상황: 식후 2h 혈당 220 mg/dL 측정**

- **당뇨 전문 내과의**: *"식후 PPG 220은 권장 범위(180 이하) 초과입니다. 식이 시퀀스를 점검하고, 반복되면 약 조정이 필요할 수 있어요."*
- **당뇨 교육 간호사**: *"오늘 식후 혈당이 조금 높았네요. 괜찮아요, 함께 원인을 찾아봐요. 혹시 식사 시 채소를 먼저 드셨나요?"*
- **혈당 코치**: *"식후 2h 220 → 어제 동시간 175 대비 +45. 식사 변동성 분석 중. 메뉴 자동 비교 가능해요."*
- **GLP-1 약물 동반자**: *"오늘 주사 시간이 평소보다 늦었나요? GLP-1 효과 피크 시간과 식사 시점 정렬을 확인해보세요."*

### 17.2 함수 시그니처 변경

**기존** (단일 프로필):
```javascript
calculateIMEM({ checks, riskActive, recoveryDone, profile, sunset, meals })
```

**신규** (프로필 인자):
```javascript
calculateIMEM({
  checks, riskActive, recoveryDone, profile, sunset, meals,
  calibrationProfile: 'obesity' | 'diabetes',  // 신규
  cgmData: { ... } | null,                      // 당뇨 전용
  medications: [ ... ] | null,                  // 당뇨 전용
})
```

### 17.3 마이그레이션 정책

기존 IMEM 사용자(인크레티나 i)는 영향 없음:
- 기본값 `calibrationProfile: 'obesity'` 적용
- 신규 인자 미전달 시 기존 동작 100% 유지

신규 IMEM-DM 브랜드는 명시적 호출:
- `calibrationProfile: 'diabetes'`
- CGM 연동 시 `cgmData` 전달
- 약물 정보 `medications` 전달

### 17.4 데이터베이스 스키마 변경

Firestore 기존 컬렉션은 유지, 당뇨 전용 컬렉션 추가:

```
users/{uid}/
  ├── (기존)
  │   ├── dailyRoutines/
  │   ├── events/
  │   └── messages/
  │
  ├── (신규 — IMEM-DM 사용자만)
  │   ├── glucoseMeasurements/   // 자가측정 + CGM
  │   │   └── {YYYY-MM-DD}/
  │   │       └── readings/
  │   ├── medications/           // 처방 정보
  │   ├── hba1c/                 // 분기 검사
  │   ├── doctorReports/         // 자동 생성 보고서
  │   └── safetyAlerts/          // 안전 이벤트 로그
```

## 18. 임상 검증 디자인 차이

### 18.1 비만 임상 vs 당뇨 임상

| 차원 | 비만 임상 | **당뇨 임상** |
|------|--------|----------|
| **1차 endpoint** | 체중 변화 (kg, %) | **HbA1c 변화** |
| **2차 endpoint** | BMI, 체지방률, 허리둘레 | TIR, CV, GMI, FPG |
| **기간** | 12-24주 | **12주 (단기) ~ 6개월 (장기)** |
| **샘플 수** | 60-100명 | **100-200명** |
| **대조군** | 일반 칼로리 코칭 | **표준 당뇨 교육** |
| **윤리심의** | IRB | IRB + 의료기기 심의 (장기) |
| **비용** | $100K-300K | **$300K-1M** |

### 18.2 IMEM-DM 임상 디자인 (제안)

**Phase 1 — Pilot (n=20-30, 8주)**
- 목적: 안전성·UX 검증
- 디자인: 단일군 관찰
- 측정: TIR, IMEM 점수, 사용자 만족도

**Phase 2 — RCT (n=120, 12주)**
- 목적: 유효성 1차 검증
- 디자인: 무작위배정 1:1 비교 (표준 당뇨 교육 vs IMEM-DM)
- 1차 endpoint: HbA1c 변화 (12주)
- 2차 endpoint: TIR, CV, 약 의존도

**Phase 3 — Pivotal (n=300+, 6개월)**
- 목적: SaMD Class II 인증 패키지
- 디자인: 다기관 RCT
- Endpoint: HbA1c, TIR, 합병증 발생률

## 20. 부록: 변화 요약 매트릭스

### 20.1 한눈에 보는 변화 매트릭스

| 차원 | IMEM (비만) | **IMEM-DM (당뇨)** | 변화 유형 |
|------|------------|-------------------|--------|
| **공식** | α × β × γ | α × β × γ × δ × ε × ζ | **확장** |
| **계수 수** | 3개 | 6개 | **확장** |
| **모드** | 단일 | Basic / Standard / Full | **신규** |
| **α 의미** | 일주기 → 지방 연소 | 일주기 → 새벽현상 + 인슐린 | **재해석** |
| **β 범위** | 0.95~1.025 | 0.85~1.05 | **확장** |
| **β 입력** | 시퀀스 | + GI, GL, 섬유 | **확장** |
| **γ 가중치** | 1.0 | 1.5 | **재조정** |
| **δ** | — | Glycemic Variability | **신규** |
| **ε** | — | 약물 보정 | **신규** |
| **ζ** | — | 스트레스 보정 | **신규** |
| **루틴 수** | 10개 | 17개 + 분기 4개 | **확장** |
| **위험 수** | 8개 | 15개 | **확장** |
| **위험 페널티** | -4 ~ -20 | **-6 ~ -25** | **재조정** |
| **Bio-Sync** | 메타볼릭 스위치 (3-4 마커) | 혈당 시계 (8-10 마커) | **확장** |
| **알림 슬롯** | 13개 | 18-22개 | **확장** |
| **AI 코치 페르소나** | 3종 | + 4종 (총 7종) | **확장** |
| **출력 지표** | IMEM 점수, 효율, 체중 예측 | + TIR, GMI, AGP, 약 의존도, 합병증 위험 | **확장** |
| **안전 레이어** | 없음 | **필수** | **신규** |
| **CGM 통합** | 없음 | **핵심 기능** | **신규** |
| **의사 보고서** | 없음 | **자동 생성** | **신규** |
| **데이터 채널** | 식사·운동·수면·체중 | + 혈당, HbA1c, 복약, HRV, 자기보고 | **확장** |
| **임상 endpoint** | 체중 변화 | HbA1c 변화 | **재정의** |

### 20.2 특허 명세서 작성 시 강조할 핵심 신규 요소

다음 12개 요소가 IMEM-DM의 **특허 청구 핵심**이 되어야 합니다:

1. **δ 계수** — Glycemic Variability를 IMEM 효율 공식에 통합
2. **ε 계수** — 약물 효과를 행동 효과와 분리하여 IMEM 점수에 반영
3. **ζ 계수** — 스트레스·HRV를 인슐린 감수성 보정 변수로 통합
4. **3단계 모드** (Basic/Standard/Full) — 데이터 가용성 기반 점진적 활성화
5. **혈당 시계 (Bio-Sync DM)** — 24h 곡선 정렬 + 8-10개 골든 마커
6. **안전 레이어** — 권고 생성 후 안전 필터 통과 정책
7. **PPG 기반 사후 보정** — 측정 혈당으로 β_meal 사후 검증·보정
8. **GLUT4 골든 윈도우** — 식후 15-45분 운동 시점 정밀화
9. **약 의존도 분리 표시** — 행동 점수 vs 약물 점수 시각화
10. **신규 위험 R-09~R-15** — 새벽현상, 저혈당, 약물 누락 등
11. **Calibration Profile 시스템** — 단일 코어, 다중 적응증 지원
12. **의사 보고서 자동 생성** — AGP + IMEM 점수 추세 통합 PDF

### 20.3 종래기술 차별화 포인트 (특허용)

특허 출원 시 다음 종래기술 대비 차별점이 명확해야 함:

| 종래기술 | 한계 | IMEM-DM 차별점 |
|--------|------|-------------|
| 단순 혈당 로깅 앱 | 행동 변화 메커니즘 부재 | α/β/γ/δ/ε/ζ 6계수 정량 엔진 |
| CGM 보조 앱 | 데이터 표시 위주 | 행동→혈당 인과관계 정량화 + AI 코치 |
| 일반 당뇨 교육 앱 | 일반론, 개인화 부족 | 시간 단위 정밀 피드백 + 페르소나 4종 |
| 약물 효과만 추적 | 행동·약물 통합 시점 부재 | ε 계수로 행동·약물 분리 정량화 |
| 일주기 기반 앱 | 당뇨 특화 부재 | 새벽현상·인슐린 일중 효율 정밀 모델링 |
